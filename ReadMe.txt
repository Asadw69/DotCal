## 🛠 Tech Stack

### Core Languages

* **TypeScript** — primary language
* **JavaScript** — configuration files (e.g. `next.config.mjs`)
* **CSS** — styling via Tailwind CSS
* **SQL** — database schema (`01-create-tables.sql`)

### Framework

* **Next.js 16.0.10** — React framework using the App Router

### UI Library

* **React 19.2.0**
* **React DOM 19.2.0**

### Styling

* **Tailwind CSS 4.1.9** — utility-first CSS framework
* **PostCSS 8.5** — CSS processing
* **Autoprefixer 10.4.20** — vendor prefixing
* **tailwindcss-animate 1.0.7** — animation utilities

### UI Components

* **Radix UI** — headless UI components (dialogs, dropdowns, tooltips, etc.)
* **shadcn/ui** — component system built on Radix UI
* **Lucide React** — icon library

### Forms & Validation

* **React Hook Form 7.60.0** — form state management
* **Zod 3.25.76** — schema-based validation
* **@hookform/resolvers 3.10.0** — Zod integration for React Hook Form

### Date & Time

* **date-fns 4.1.0** — date utilities
* **react-day-picker 9.8.0** — calendar component

### State & Storage

* **localStorage** — client-side persistence
* **Supabase (optional)** — backend and SSR support

  * `@supabase/supabase-js`
  * `@supabase/ssr`

### Utilities & Other Libraries

* **next-themes 0.4.6** — dark/light mode support
* **class-variance-authority** — component variant management
* **clsx** & **tailwind-merge** — conditional class utilities
* **sonner** — toast notifications
* **recharts** — charts and data visualization
* **vaul** — drawer component
* **cmdk** — command menu

### Development Tools

* **TypeScript 5** — static type checking
* **ESLint** — linting and code quality

---

## 📌 Summary

A **Next.js (React) application written in TypeScript**, styled with **Tailwind CSS**, using **Radix UI / shadcn/ui** for accessible components, **React Hook Form + Zod** for form validation, and **localStorage** (with optional **Supabase**) for data persistence.
